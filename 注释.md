# 注释
## 单行注释
```
1. 单行注释以两个斜线开始，以行尾结束
 // 这是一句单行注释
2. 使用方法
    * 独占一行的注释，用来解释下一行代码(注释前有一个空行，并且缩进与下一行代码保持一致)
    * 在代码行的尾部注释（代码注释到结束之间至少有一个缩进，不能超过最大字符数限制）
    * 被注释掉的大段代码
```
## 多行注释
```
1. 多行注释以/*开始，以*/结束。
2. 规范： 
    /*
     *另一段注释
     */
3. 多行注释总是出现在要描述的代码段之前，注释和代码之间没空行间隔。
```
## 使用注释
 ```
 添加注释的原则是在需要让代码更清晰时添加注释:
 1. 难以理解的代码
 2. 可能被理解为错误的代码
 3. 浏览器特性hack
 ```
## 文档注释
```
对文档进行注释：
    1. 所有的方法：应当对方法、期望参数和可能的返回值添加注释描述 
    2. 所有的构造函数：应当对自定义类型和期望的参数添加注释描述
    3. 所有包含文档化方法的对象

    /**
     @method merge
     @param { Object } 被合并的一个或多个对象
     @return { Object }   
     **/
```
