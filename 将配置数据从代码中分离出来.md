# 将配置数据从代码中分离出来
## 1. 什么是配置数据
```
配置数据是应用中写死的值。
    > URL
    > 需要展现给用户的字符串
    > 重复的值
    > 设置(比如每页的配置项)
    > 任何可能发生变更的值
```
## 2. 抽离配置数据
```
将配置数据从代码中抽离出的第一步是将配置数据拿到外部，即将数据从JavaScript代码中拿掉。
    > var config = {
        MSG_INVALID_VALUE: "Invalid value",
        URL_INVALID: "/errors/invalid.php",
        CSS_SEL: "select"
    }
```
## 3. 保存配置数据
```
配置数据最好放到单独的文件中，以便清晰地分割数据和应用逻辑。
```
