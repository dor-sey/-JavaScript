# 文件和目录结构
## 1. 最佳实践
```
1. 一个文件只包含一个对象
    >特制每个JavaScript文件应该只包含一个JavaScript对象的代码。
2. 相关的文件用目录分组
    >如果有多个相连的对象，就把这些文件都放在同一个目录下
3. 保持第三方代码的独立
    >引入一个JavaScript类库最好直接用CDN来加载或者将这些文件单独放在项目工程的一个目录。
4. 确定创建位置
    >创建后的JavaScript文件应该放在一个单独的文件里。
5. 保持测试代码的完整性
```
## 2. 基本结构
```
> build：用来放置最终构建的文件，一般这个目录不会提交
> src：用来放置所有的源文件
> test或tests：放置所有的测试文件，通常包含一些同源代码相同的目录。
```
