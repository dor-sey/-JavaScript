# 基本的格式化
## 缩进层级
```
1.使用制表符进行缩进
2.使用空格符进行缩进(2、4、8个空格)

推荐使用4个空格符缩进
```
## 语句结尾
```
使用;结尾
```
## 行的长度
```
JavaScript代码单行长度不超过80个字符(没有明确规范)
```
## 换行
```
1. 通常换行下一行会增加两个层级的缩进,避免在运算符处换行
    //好的做法：
    callAfunction(document, element,
            navigator);
2. 给变量赋值时，第二行的位置应和赋值运算符保持对齐
    //例:
    var resutl = something + anotherthing +
                 finalthing;   
```
## 空行
```
合理的应用空行,能更流畅地阅读代码：
* 在方法之间
* 在方法的局部变量和第一条语句之间
* 再多行或单行注释之前
* 在方法内的逻辑片段之前插入
```
## 命名
```
一般使用小驼峰命名；

1.变量和函数
    * 变量前缀应当是名词
    * 函数前缀应当是动词(例：can、has、is、get、set、do等)
2. 常亮
    * 常亮应当使用大写字母和下划线来命名// var MAX_COUNT = 10;
3. 构造函数
    * 构造函数应当使用大驼峰命名法
```
## 直接量
```
1. 字符串
    * 字符串可以使用单引号或双引号
    * 多行字符串应当书写规范
        //不好的写法
        var longString = "Here is the \
        story";
        //规范写法
        var longString = "Here is the "+
                        "story";

2. 数字   
    * 不推荐八进制写法 //var num = 010; //数字010表示不是十进制的10，而是8，容易造成错误
3. null
    * 作为对象的占位符
4. undefined
5. 对象直接量
    * 推荐使用对象直接量来创建对象:var a = { title: 'js'}
6. 数组直接量
    *推荐使用数组直接量来创建数组：var nums = [ 1, 2, 3 ]

```
